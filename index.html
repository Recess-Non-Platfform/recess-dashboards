Steps

  1. Open Airtable token page: Go to https://airtable.com/create/tokens (log in if prompted)
  2. Click the blue "Create new token" button (top right)
  3. Name the token: Type Steve LinkedIn Reader in the Name field
  4. Add a scope: Click "Add a scope" → select data.records:read (this gives read-only access — Steve can't modify anything)
  5. Add a base: Click "Add a base" → search for and select your Fireflies transcripts base (the one where meeting recordings/transcripts land)
  6. Click "Create token" → a token starting with pat... appears. Copy it now — Airtable only shows it once
  7. Get the Base ID: Open the Fireflies transcripts base in Airtable. Look at the URL bar — it looks like https://airtable.com/appABC123XYZ/tblXXX/.... Copy the part that starts with app (e.g. appABC123XYZ)
  8. Save both values to the env file. Open Terminal and run:
    nano ~/clawd/Projects/linkedin-content-writer/.env
    Paste these two lines (replacing the placeholder values):
    AIRTABLE_PAT=pat_your_token_here
    AIRTABLE_BASE_ID=appYour_base_id_here
    Press Ctrl+O then Enter to save, then Ctrl+X to exit
  9. Verify: Run cat ~/clawd/Projects/linkedin-content-writer/.env — you should see your two values (not the placeholders)
