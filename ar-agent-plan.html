Architecture

Task-Based State Machine

Every invoice is a managed task with its own lifecycle. Tasks determine their own next action based on state — no cron jobs blindly firing.

INVOICE LIFECYCLE

NEW → SCHEDULED → DRAFT_READY → APPROVED → SENT
REVISING human editing SNOOZED deferred
↓ 7-day suppression window / monitor ↓
REPLY_RECEIVED → RESOLVED
ESCALATE → WAITING / PARTIAL_PAYMENT
↓ "Check is in the mail" / payment promised ↓
PAYMENT_CLAIMED 7-10 biz day verification → RESOLVED if payment posts
window expires → RE-ESCALATE 2nd miss → flag stall tactic → Deuce
BOUNCED email delivery failed CANCELLED / WRITTEN_OFF terminal states

Why Tasks > Cron

  • ✓ Each invoice has state — last action, suppression window, escalation level
  • ✓ Tasks determine their own next action based on current state
  • ✓ Tasks persist across sessions — no amnesia
  • ✓ No double-sends, no missed follow-ups
  • ✓ Full audit trail for every action taken

🧾 PAYMENT_CLAIMED State

When a customer says "check is in the mail" → enter PAYMENT_CLAIMED

  • ✓ Set 7-10 business day verification window
  • ✓ Monitor QBO for payment posting
  • ✓ If payment posts → auto-close, send thank you
  • ✓ If window expires → re-escalate with context ("You mentioned on [date] a check was mailed...")
  • ✓ If 2nd miss → flag as stall tactic → escalate to Deuce

The Chobani pattern — they claimed they sent checks but nothing posted.

Risk-Scored Actions

Within each phase, actions are gated by a risk score:

risk = balance × overdue_days × action_type_weight

  • Low risk: Pre-due reminder on $500 invoice → auto-advances faster
  • Medium risk: Firm follow-up on $8K → standard phase rules
  • High risk: Escalation on $21K → always needs human approval

Contact Resolution Priority

Karen resolves the best contact for each invoice using this priority chain:

  1. QBO billing_email — actual invoice recipient (filter out @recess.is)
  2. QBO billing_cc_email — CC'd external contacts
  3. HubSpot AP Invoice Email — deal-level billing contact
  4. HubSpot Deal Contact — primary relationship contact
  5. Deal Associated Contacts — additional contacts on the deal
  6. Account Manager — HubSpot am_lead field (CC on all emails, escalation path)

Coverage: 19/21 customers (90%) fully linked. Only Danone ($50) and Ramp ($250) unlinked — both trivial.

Smart Suppression Rules

💬 Customer replied in last 7 days → SKIP, create next-action task for +7d
✅ Payment detected (balance decreased) → auto-close + thank-you draft
💵 Partial payment → acknowledge, adjust tone, update balance
📦 3+ invoices same deal → consolidate into ONE email
📨 Incoming email detected → flag for HUMAN response (they asked us something)
🤖 Known question (W9/PO/remittance) → auto-draft standard response
⏰ Contact hasn't appeared in 6+ months → flag as "possibly stale"
